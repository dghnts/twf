# スペックドキュメント形式の指定

スペックモードで requirements.md と design.md を作成する際は、以下の形式に従ってください。

## requirements.md の形式

requirements.md は以下のセクション構成で作成してください：

### 1. 機能概要
- この機能の目的と概要を簡潔に記述
- 何を実現するのかを明確に説明

### 2. 機能要件
- 実装すべき具体的な機能をリスト化
- 各機能要件は明確で測定可能な形で記述

### 3. 非機能要件
- パフォーマンス、可用性、保守性などの要件
- 品質特性に関する要件を記述

### 4. 制約事項
- 技術的制約、環境制約、リソース制約など
- 開発・運用上の制限事項を明記

## design.md の形式

design.md は以下のセクション構成で作成してください：

### 1. 処理一覧
- 実装する処理の一覧を表形式または箇条書きで記載
- 各処理の名称と簡単な説明

### 2. 処理詳細
- 各処理の詳細な実装方法を記述
- アルゴリズム、フロー、入出力などを明確に説明

### 3. データモデル
- 使用するデータ構造、テーブル定義、エンティティなどを記述
- 必要に応じて図や表を使用

### 4. API設計
- エンドポイント、リクエスト/レスポンス形式を定義
- APIが存在しない場合は、関数インターフェースや公開メソッドを記述

## 記述ルール

- すべて日本語で記述
- 各セクションは見出しで明確に区切る
- 具体的で実装可能な内容を記述
- 必要に応じてコードブロック、表、箇条書きを活用
